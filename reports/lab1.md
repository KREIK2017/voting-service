# ЛАБОРАТОРНА РОБОТА № 1

**Тема роботи:** Створення Product Backlog.

**Мета роботи:** Придбання навичок створення Product Backlog.

## Завдання

1) створити Scrum команду (4 людини);

2) визначитись з ролями кожного учасника. Ролі гібридні: Scrum майстер – розробник; розробник – front-end; розробник – back-end; розробник – тестувальник (ролі гібридні, а це означає, що у разі необхідності людина може працювати не у своїй ролі. Окрім того кожний відповідає за інтеграцію технологій);

3) визначити тематику розробки;

4) створити Product backlog (оцінки важливості та складності виставити прибизні);

5) на основі складеного Product backlog скласти Sprint Backlog для першого Sprint (враховуючи, що Sprint триватиме два тижні).

## Виконання завдання

### Scrum команда

| Учасник | Основна роль | Гібридна роль |
|---|---|---|
| Учасник 1 | Scrum майстер | Back-end (Laravel) |
| Учасник 2 | Back-end (Laravel, MySQL) | Інтеграція API |
| Учасник 3 | Front-end (Blade, Bootstrap 5, Next.js) | UI / UX |
| Учасник 4 | Тестувальник | Документація |

### Тематика розробки

**Сервіс онлайн-голосувань** — веб-застосунок для створення опитувань та збору голосів. Складається з двох частин: full-stack Laravel-застосунку з Blade-фронтендом і Next.js SPA, що підключається через REST API. Передбачає дві ролі (адміністратор створює та керує опитуваннями; виборець голосує та переглядає результати), email-верифікацію, локалізацію (українська / англійська), API через Laravel Sanctum.

**Технологічний стек:** PHP 8.2, Laravel 13, MySQL (XAMPP), Bootstrap 5, Next.js 15 (TypeScript), Laravel Sanctum.

## Висновки

У ході виконання лабораторної роботи було створено команду і визначено ролі. Складено Product Backlog для проєкту сервісу онлайн-голосувань, який включає всі необхідні завдання для реалізації ключових функціональних вимог: автентифікацію з email-верифікацією, авторизацію за двома ролями (admin / voter), CRUD опитувань та варіантів відповідей, логіку голосування з підрахунком результатів, REST API та Next.js фронтенд. На основі цього списку було створено Sprint Backlog для першого спринту, що триватиме два тижні. Основний акцент першого спринту зроблено на стадії проєктування, яка включає: створення діаграм сценаріїв використання, що визначають основні потоки взаємодії користувачів (адміністратор / виборець) із системою; побудову діаграми класів для опису логічної структури проєкту (моделі User, Poll, Option, Vote); проєктування бази даних для зберігання інформації про користувачів, опитування, варіанти та голоси; створення графічного інтерфейсу ключових сторінок; налаштування середовища розробки (Laravel 13, XAMPP, MySQL).
